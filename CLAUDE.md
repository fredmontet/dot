# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Personal macOS dotfiles, organized with a topic-based ("relationships") structure inspired by [Holman's dotfiles](https://github.com/holman/dotfiles). Cloned to `~/.dot` and referenced as `$DOT` from the shell. This is configuration, not an application — there is no build/test/lint suite.

## Commands

- `make setup` (= `run/setup`) — Symlink all `*.symlink` files into `$HOME` and generate the local git config. Idempotent; prompts skip/overwrite/backup on conflicts.
- `make install` (= `run/install`) — Run every `rel/*/install.sh` (Homebrew bundle, macOS updates, …).
- `make all` — `setup` then `install`.
- `dot` — Periodic refresh: runs `rel/os/reset.sh`, `brew upgrade`, then `run/install`. `dot -e` opens the repo in `$EDITOR`; `dot -h` for help. On `$PATH` via `rel/os/path.zsh`.
- `reload` — Re-source `~/.zshrc` (alias in `rel/zsh/aliases.zsh`).

## Architecture

### Topic directories (`rel/<topic>/`)
Each subdirectory of `rel/` is a self-contained "relationship" holding the config for one tool/concern (`git`, `zsh`, `brew`, `os`, `anthropic`, …). There is no central registry — files are picked up by convention based on extension/name. To add a tool, create `rel/<name>/` and drop in the relevant files.

### File conventions (the core mechanism)
These conventions, enforced across `rel/zsh/zshrc.symlink`, `run/setup`, and `run/install`, are what make the repo work:

- **`*.zsh`** — Auto-sourced into the shell. `zshrc.symlink` globs `$DOT/**/*.zsh` and sources them in three passes: `path.zsh` first, then everything except path/completion, then `completion.zsh` last. **Any `.zsh` file anywhere under `rel/` loads automatically** — ordering is by filename convention, not directory.
- **`*.symlink`** — `run/setup` links each `foo.symlink` → `~/.foo` (basename, extension stripped, dot-prefixed; `find -maxdepth 3`).
- **`install.sh`** — Run by `run/install` (`find ./rel -name install.sh`). Deliberately `.sh`, not `.zsh`, so it does NOT auto-load into the shell — it only executes during install.
- **`path.zsh`** — Sourced first; sets up `$PATH`/env. `rel/os/path.zsh` is what puts `$DOT/bin` on `$PATH`.

### `bin/`
Scripts here are on `$PATH` everywhere (via `rel/os/path.zsh`). Currently just `dot` (the refresh command).

### Secrets & machine-local config (all gitignored)
- `~/.localrc` — sourced early by `zshrc.symlink`; machine-local env vars / secrets go here, never in the repo.
- `rel/git/gitconfig.local.symlink` — generated by `run/setup`'s `setup_git()` from `gitconfig.local.symlink.example` (interactive prompts → `sed`), linked to `~/.gitconfig.local`, and `[include]`d by `gitconfig.symlink` (which holds no identity itself).
- `**/env.zsh` — per-topic secrets (e.g. `rel/anthropic/env.zsh` exports `ANTHROPIC_API_KEY`); auto-sourced like any `.zsh` but gitignored. Commit a `env.zsh.example` breadcrumb instead of the real file. `rel/brew/Brewfile.lock.json` is also gitignored.

## Notes
- Homebrew packages live in `rel/brew/Brewfile`; `rel/brew/install.sh` runs `brew bundle`.
- macOS-specific throughout: `osxkeychain`, `softwareupdate`, casks, `pbcopy`, `gls`.
- The bottom of `zshrc.symlink` holds machine-appended blocks written by tool installers (Fig pre/post, iTerm2, Antigravity, Windsurf). Leave the Fig pre block at the very top and post block at the very bottom.
