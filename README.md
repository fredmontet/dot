# .dot

[![lint](https://github.com/fredmontet/dot/actions/workflows/lint.yml/badge.svg)](https://github.com/fredmontet/dot/actions/workflows/lint.yml)

This repository groups my different 'dot files' with a structure inspired from Zack Holman (https://github.com/holman/dotfiles). Following is an explanation of how my dotfiles work.

## Concept

Everything's built around **relationships**. If you're adding a new relation to your dotfiles, say "python", you can simply add a `python` directory in the `rel` folder and put files in there.

Anything with an extension of `.zsh` will get automatically included into your shell. Anything with an extension of `.symlink` will get symlinked without extension into `$HOME` when you run the `setup` script.

## Main Structure

- `bin/`: Anything in `bin/` will get added to your `$PATH` and be made
  available everywhere.
- `rel/`: All relationships go in there.
- `run/`: All scripts go in there.

### Special files in \<relation>

- `<relation>/*.zsh`: Any `.zsh` file gets loaded into your environment.
- `<relation>/path.zsh` is loaded first and is expected to setup `$PATH` or similar.
- `<relation>/completion.zsh` is loaded last — autocomplete, plus any plugins that
  must run after `compinit`.
- `<relation>/install.sh` is executed when you run `run/install`. To avoid being
  loaded automatically, its extension is `.sh`, not `.zsh`.
- `<relation>/*.symlink`: Any file ending in `*.symlink` gets symlinked into
  your `$HOME` (e.g. `gitconfig.symlink` → `~/.gitconfig`).
- `<relation>/env.zsh`: Secrets / machine-local env vars. Loaded like any `.zsh`
  but **gitignored** — commit an `env.zsh.example` template instead, which
  `setup` scaffolds into the real file on a fresh machine.

### Scripts

Run them directly (`run/<name>`) or via `make`:

- `setup` (`make setup`): symlink everything, generate your local git config,
  link `rel/ssh/config` → `~/.ssh/config`, and scaffold any `*.example` files.
- `install` (`make install`): run every `rel/*/install.sh` (Homebrew bundle,
  macOS updates, …).
- `lint` (`make lint`): `shellcheck` the scripts and `zsh -n` the shell config.
- `make all`: `setup` then `install`.

Linting also runs in CI on every push (see `.github/workflows/lint.yml`).

## Get Started

```sh
# Install homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install the dotfiles
git clone https://github.com/fredmontet/dot.git ~/.dot
cd ~/.dot
make setup     # symlink dotfiles into $HOME, set up git/ssh
make install   # install Homebrew packages & tools (or run `make all`)
```

This will symlink the appropriate files in `.dot` to your home directory.
The main file you'll want to change right off the bat is `rel/zsh/zshrc.symlink`,
which sets up a few paths that'll be different on your particular machine.

## Machine-local config & secrets

Anything private or machine-specific stays out of this (public) repo:

- `~/.localrc` — sourced early by `zshrc.symlink`; put env vars / secrets here.
- `**/env.zsh` — per-topic secrets (e.g. `rel/anthropic/env.zsh`); gitignored,
  with a committed `env.zsh.example` template.
- `~/.gitconfig.local` — your git identity, generated by `setup` from
  `rel/git/gitconfig.local.symlink.example`.

## Useful Commands

- `dot` pulls the latest dotfiles, upgrades Homebrew, re-applies macOS defaults,
  and re-runs the installers — run it now and then to keep your environment fresh.
  `dot -e` opens the repo in `$EDITOR`, `dot -h` shows help. It lives in `bin/`.
- `reload` re-sources `~/.zshrc`.
- `make lint` static-checks the scripts and shell config.

The interactive shell comes with fuzzy search (`fzf`), smart `cd` (`zoxide`),
autosuggestions and syntax highlighting.
